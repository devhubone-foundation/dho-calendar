# @dho/database

Prisma schema, migrations, and seed data shared by `apps/api`.

## Migration convention

Migrations use Prisma's default timestamped folder format:
`prisma/migrations/<UTC timestamp>_<snake_case description>/migration.sql`, generated by:

```bash
pnpm --filter @dho/database prisma:migrate --name <description>
```

Rules for downstream issues:

- Never edit a committed migration folder (including `..._init`, added by Issue #1).
- Always add a **new** migration folder for schema changes.
- Name the migration after the owning feature, e.g. `_add_office_schedule`, `_add_events`.
- Additive changes only where possible; coordinate destructive changes (column/type removal) via the shared-file rule in `CONTRIBUTING.md`.

## Local commands

```bash
pnpm --filter @dho/database db:generate          # regenerate the Prisma client
pnpm --filter @dho/database prisma:migrate       # create/apply a dev migration (needs a shadow DB)
pnpm --filter @dho/database prisma:migrate:deploy # apply committed migrations only (CI/CD)
pnpm --filter @dho/database prisma:seed          # seed one admin + one temp-password member
```
